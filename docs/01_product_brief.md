# Colibri Chat
**Instant, small, and soon gone.**

Colibri Chat is a proximity-based, location-bound, time-bound chat system for adults (18+) to coordinate in the moment with people physically nearby.  
Core principle: **ephemeral + contextual + safe beats engagement**.

This repo contains:
- **Supabase/Postgres** schema, RLS policies, and **Postgres RPC** functions
- Supabase Edge Functions (thin wrappers) for join/heartbeat/send-message
- Docs/specs to drive a coding agent (Claude Code/Codex) to build the mobile client

> Notes aligned with the product concept: proximity rooms, ephemeral messages, presence-based UI (grey when sender is no longer nearby), strict safety & privacy constraints. :contentReference[oaicite:1]{index=1}

## v0 Scope (MVP)
✅ Included:
- Neighborhood rooms + Event rooms
- Location permission required to join/post
- Public group chat only (**NO DMs in v0**)
- Message length <= 200 chars
- Rate limit: 1 message / 5 seconds (server enforced)
- Neighborhood messages expire after **60 minutes**
- Event messages expire at **min(created_at + 60m, event ends_at)**
- Presence greying: messages greyed when sender is outside geofence or stale heartbeat
- Reporting & shadow mute threshold
- Strict PII sharing restriction in public chat (phone/email/social handles blocked)

🚫 Not included (explicit non-goals v0):
- DMs / friend/follow
- Persistent profiles, usernames
- Reactions/likes
- Search / message history beyond TTL
- Growth loops

## Geofence Rules
- Target radius: **100 ft** with **±20 ft tolerance**
- We model this as:
  - `radius_m = 30`
  - `tolerance_m = 6`
  - allow distance <= radius_m + tolerance_m
- Post gate: location accuracy must be <= **25m** (configurable)

## Tech Stack (v0)
- Mobile: Expo / React Native (to be generated by the coding agent)
- Backend: Supabase (Postgres + RLS + Realtime)
- Geo: PostGIS (geography)
- Business logic: Postgres RPC functions (primary) + Edge Functions wrappers (optional)

## Setup (Supabase)
1) Create a Supabase project
2) Enable extensions (migration does this): `postgis`, `pgcrypto`
3) Apply migrations:
   - Run `supabase db push` (if using Supabase CLI)
   - Or paste `supabase/migrations/001_init.sql` into SQL Editor

4) Apply RLS policies:
   - Run `supabase/policies/rls.sql` in SQL Editor

## Definition of Done
v0 is “done” when all acceptance tests in `docs/07_acceptance_tests.md` pass and the following works end-to-end:
- user can see nearby rooms, join, read/post messages
- geofence + accuracy gates are enforced server-side
- messages expire as specified
- reports lead to shadow mute
- presence greying works via heartbeat
